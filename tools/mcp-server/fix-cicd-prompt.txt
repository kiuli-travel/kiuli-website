## Step 4: Diagnose Vercel auto-deploy

This is the critical step. We need to determine if Vercel's GitHub integration is connected and auto-deploying on push.

### 4a: Check Vercel project linking
```bash
cat .vercel/project.json
```

This shows the project ID and org ID. Note them.

### 4b: Check GitHub integration
```bash
vercel git ls
```

This should show which GitHub repo is connected and which branch triggers production deploys. If this command fails or shows no integration, that is the root cause.

### 4c: List recent deployments
```bash
vercel ls --prod --limit 5
```

Note the most recent deployment URL, its state, and when it was created. Compare the timestamp to the most recent git push. If the most recent deployment is older than the most recent push, auto-deploy is broken.

### 4d: Inspect the latest deployment
Take the URL from 4c and run:
```bash
vercel inspect <deployment-url>
```

This shows what commit it was built from. Compare to HEAD.

### 4e: Report findings

Print a summary:
```bash
echo "=== CI/CD DIAGNOSIS ==="
echo "Local HEAD: $(git rev-parse --short HEAD)"
echo "Remote HEAD: $(git ls-remote --heads origin main | cut -c1-7)"
echo "Latest Vercel deployment commit: <from step 4d>"
echo "GitHub integration status: <from step 4b>"
echo "Auto-deploy working: YES/NO"
```

