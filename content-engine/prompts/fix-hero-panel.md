# Fix: Hero Image Rendering and Hero Video Picker Removal

Two independent fixes. Make both changes, then build once at the end.

---

## Fix 1 — Hero image not rendering in HeroPanel

**File:** `src/components/admin/editorial/HeroPanel.tsx`

**Problem:** When `hasImage` is true, the component renders a solid `<div>` with `backgroundColor: "#C8C0B0"` but no `<img>` tag. The `heroImageUrl` prop is never used to display the actual image.

**Change:** Replace the `hasImage` branch of the image preview with an actual `<img>` element.

Find this block (lines ~167–186):
```tsx
{hasImage ? (
  <div
    className="relative overflow-hidden rounded-md"
    style={{
      width: 200,
      height: 133,
      backgroundColor: "#C8C0B0",
    }}
  >
    {isImageLocked && (
      <div
        className="absolute top-1.5 left-1.5 flex items-center gap-1 rounded-full px-1.5 py-0.5"
        style={{
          backgroundColor: "rgba(0,0,0,0.5)",
          color: "#FFFFFF",
          fontSize: 10,
        }}
      >
        <Lock className="h-2.5 w-2.5" />
        <span>Locked</span>
      </div>
    )}
  </div>
```

Replace with:
```tsx
{hasImage ? (
  <div
    className="relative overflow-hidden rounded-md"
    style={{
      width: 200,
      height: 133,
      backgroundColor: "#C8C0B0",
    }}
  >
    <img
      src={heroImageUrl!}
      alt={heroImageAlt}
      style={{
        width: "100%",
        height: "100%",
        objectFit: "cover",
        display: "block",
      }}
    />
    {isImageLocked && (
      <div
        className="absolute top-1.5 left-1.5 flex items-center gap-1 rounded-full px-1.5 py-0.5"
        style={{
          backgroundColor: "rgba(0,0,0,0.5)",
          color: "#FFFFFF",
          fontSize: 10,
        }}
      >
        <Lock className="h-2.5 w-2.5" />
        <span>Locked</span>
      </div>
    )}
  </div>
```

## Fix 2 — Hero video: remove picker, display scraped video only

The hero video is generated by iTrvl for the specific itinerary. The scraper populates it. Designers should not be able to browse a library of videos from other itineraries. Remove the picker entirely.

### 2a — Update HeroPanel.tsx

**Remove `onSelectVideo` from the props interface.** The interface currently has:
```tsx
  onSelectVideo: () => void
```
Delete that line.

**Remove `onSelectVideo` from the destructured props** in the function signature.

**In the expanded video section**, find the "Select Video Button" block:
```tsx
              {/* Select Video Button */}
              <div>
                <button
                  type="button"
                  onClick={onSelectVideo}
                  className="cursor-pointer rounded-md px-3.5 py-1.5 text-xs font-medium transition-colors"
                  style={{
                    border: "1px solid #486A6A",
                    color: "#486A6A",
                    backgroundColor: "#FFFFFF",
                  }}
                  onMouseEnter={(e) =>
                    (e.currentTarget.style.backgroundColor = "#F0F5F5")
                  }
                  onMouseLeave={(e) =>
                    (e.currentTarget.style.backgroundColor = "#FFFFFF")
                  }
                >
                  Select Video
                </button>
              </div>
```
Delete this entire block.

**In the video preview section**, replace the no-video placeholder text. Find:
```tsx
                  <span style={{ fontSize: 11, color: "#DADADA" }}>
                    No video selected
                  </span>
```
Replace with:
```tsx
                  <span style={{ fontSize: 11, color: "#DADADA" }}>
                    No video from iTrvl
                  </span>
```

**In the collapsed header**, find:
```tsx
            {hasVideo ? heroVideoName : "No video selected"}
```
Replace with:
```tsx
            {hasVideo ? heroVideoName : "No video from iTrvl"}
```

**Remove the locked checkbox from the video section.** Find and delete this block:
```tsx
                <Checkbox
                  id="video-locked"
                  checked={isVideoLocked}
                  onChange={onVideoLockedChange}
                  label="Hero Video Locked"
                  helperText="Lock to prevent auto-replacement on re-scrape"
                />
```

**Remove `isVideoLocked` and `onVideoLockedChange` from the props interface:**
```tsx
  isVideoLocked: boolean
  onVideoLockedChange: (v: boolean) => void
```
Delete both lines.

**Remove them from the destructured props** in the function signature as well.
